# CIRO Pipeline Contract

> Authoritative JSON contract for `POST /pipeline/run`. Web dashboard, Flutter app, ADK tools, and scenario runner all build against this without reading backend source.

This document describes the **exact shape** returned by the single-call endpoint that chains all five CIRO agents. If you add a field to a model, update this file in the same commit.

---

## Endpoint

```
POST /pipeline/run
Content-Type: application/json
```

### Request

```json
{
  "source": "social",
  "text": "G-10 mein pani bhar gaya hai, gaariyan phans gayi hain",
  "metadata": null
}
```

| Field | Type | Required | Notes |
|---|---|---|---|
| `source` | `string` | yes | `"social"` \| `"weather"` \| `"traffic"` |
| `text` | `string` | yes | Raw signal text. Urdu (romanised or script) or English. |
| `metadata` | `object \| null` | no | Optional. Honoured keys: `geo` (string fallback location), `city`, `severity`. |

### Response (`200 OK`)

The response is a `PipelineResult` containing six top-level keys, captured from a real run below. Each block is the output of one agent stage.

```json
{
  "run_id": "run_20260516_135733",
  "batch":      { ... SignalBatch ... },
  "event":      { ... CrisisEvent ... },
  "analysis":   { ... CrisisAnalysis ... },
  "plan":       { ... ActionPlan ... },
  "simulation": { ... SimulationResult ... },
  "total_duration_ms": 12
}
```

## Stage 1 — `batch` (Signal Ingestion)

```json
{
  "batch_id": "batch_20260516_135733_3341",
  "primary_location": "G-10",
  "signals": [
    {
      "signal_id": "sig_c5e8ca28",
      "source": "social",
      "content": "G-10 mein pani bhar gaya hai, gaariyan phans gayi hain",
      "location": "G-10",
      "timestamp": "2026-05-16T13:57:33.635612Z",
      "language": "ur",
      "severity_hint": "high",
      "keywords": ["pani", "bhar", "phans gayi"]
    }
  ]
}
```

| Field | Type | Notes |
|---|---|---|
| `batch_id` | `string` | `batch_<UTC timestamp>_<4 hex>` |
| `primary_location` | `string \| null` | Most common location across the buffer. |
| `signals[].signal_id` | `string` | `sig_<8 hex>` |
| `signals[].source` | `string` | Mirrors the incoming `source`. |
| `signals[].content` | `string` | Original text. |
| `signals[].location` | `string \| null` | Resolved Pakistani location. |
| `signals[].timestamp` | `string` (ISO 8601 UTC) | When the signal was normalised. |
| `signals[].language` | `string` | `"en"` or `"ur"`. |
| `signals[].severity_hint` | `string` | `"low"` \| `"medium"` \| `"high"`. |
| `signals[].keywords` | `string[]` | Crisis-related tokens found in the text. |

The buffer is a ring of up to 5 most-recent signals. A single `/pipeline/run` call returns the full buffer in `signals`, not just the new one.

## Stage 2 — `event` (Event Detection)

```json
{
  "event_id": "evt_1cc7b8ab",
  "crisis_type": "flood",
  "location": "G-10",
  "confidence": 0.85,
  "severity": "critical",
  "signals": [ ... same shape as batch.signals ... ],
  "explanation": "FLOOD detected at G-10 with 85% confidence from 5 signal(s).",
  "detected_at": "2026-05-16T13:57:33.644642Z"
}
```

| Field | Type | Allowed values |
|---|---|---|
| `event_id` | `string` | `evt_<8 hex>` |
| `crisis_type` | `string` (enum) | `flood`, `heatwave`, `blockage`, `accident`, `fire`, `earthquake`, `storm`, `infrastructure` |
| `confidence` | `float` | 0.0–1.0 |
| `severity` | `string` (enum) | `low`, `medium`, `high`, `critical` |
| `location` | `string` | Mirrors `batch.primary_location` or first signal's location. |

Severity-to-confidence thresholds (`confidence_to_severity`):

| confidence ≥ | severity |
|---|---|
| 0.75 | `critical` |
| 0.55 | `high` |
| 0.35 | `medium` |
| 0.00 | `low` |

## Stage 3 — `analysis` (Reasoning Agent)

```json
{
  "analysis_id": "ana_49178f26",
  "event_id": "evt_1cc7b8ab",
  "impact": [
    "Residential areas in G-10 submerged with 2-3 feet of standing water",
    "Vehicular traffic completely halted on main arteries",
    "Risk of waterborne disease outbreak within 24-48 hours"
  ],
  "affected_population": 3200,
  "infrastructure_at_risk": ["roads", "drainage", "power grid", "telecom towers"],
  "urgency": "immediate",
  "summary": "Severe flash flooding in G-10 sector requires immediate rescue and evacuation operations."
}
```

| Field | Type | Notes |
|---|---|---|
| `urgency` | `string` (enum) | `immediate` \| `within_hour` \| `monitoring` |
| `affected_population` | `int` | Estimated. May be `0` for low-severity events. |
| `impact` | `string[]` | 2–4 bullet points. |
| `infrastructure_at_risk` | `string[]` | Free-form labels (`roads`, `power grid`, etc.). |

When `GROQ_API_KEY` is set, the bullets and summary are generated by `llama-3.3-70b-versatile` via Groq. Otherwise the cached library is used. The **shape is identical** in both modes — UI code never needs to branch.

## Stage 4 — `plan` (Action Planning)

```json
{
  "plan_id": "plan_3b1bec04",
  "event_id": "evt_1cc7b8ab",
  "actions": [
    {
      "action_id": "act_46705a",
      "type": "reroute_traffic",
      "params": {
        "target_sector": "G-10",
        "crisis_type": "flood"
      },
      "priority": 1
    },
    {
      "action_id": "act_6e8617",
      "type": "dispatch_rescue_boats",
      "params": { "target_sector": "G-10", "crisis_type": "flood" },
      "priority": 2
    }
  ]
}
```

| `action.type` | Triggers in simulation |
|---|---|
| `reroute_traffic` | Drops congestion on the busiest route |
| `dispatch_rescue_boats` | Creates a ticket with unit `"Rescue Boats"` |
| `dispatch_traffic_police` | Creates a ticket with unit `"Traffic Police"` |
| `dispatch_ambulance` | Creates a ticket with unit `"Rescue 1122"` |
| `dispatch_fire_brigade` | Creates an ambulance-style ticket |
| `send_alert` / `send_flood_alert` / `send_heat_advisory` | Creates a citizen alert |
| `open_cooling_centre` / `open_cooling_centres` / `open_relief_camp` | Registers a resource |
| `update_navigation_apps` | Creates a citizen alert |
| `restrict_outdoor_activity` | Creates a citizen alert |
| `close_road_segment` | Drops congestion on busiest route |

`actions` may be empty for `low`-severity events (the planner returns `[]`).

## Stage 5 — `simulation` (Simulation Engine)

```json
{
  "run_id": "run_20260516_135733",
  "actions_executed": ["reroute_traffic", "dispatch_rescue_boats", "send_flood_alert", "open_relief_camp"],
  "tickets_created": [
    {
      "ticket_id": "tic_00bef7",
      "crisis_type": "flood",
      "location": "G-10",
      "unit_dispatched": "Rescue Boats",
      "eta_minutes": 8,
      "status": "dispatched",
      "created_at": "2026-05-16T13:57:33.644642Z"
    }
  ],
  "alerts_sent": [
    {
      "alert_id": "alr_b21cd9",
      "message": "Emergency alert: please evacuate the area.",
      "target_area": "G-10",
      "channel": "push",
      "sent_at": "2026-05-16T13:57:33.644642Z",
      "recipients_count": 2841
    }
  ],
  "routes_updated": [
    { "route": "Shahrah-e-Faisal", "before": 90, "after": 15 }
  ],
  "state_before": {
    "traffic_routes": { "G-10 to Blue Area": 85, "Shahrah-e-Faisal": 90, "Margalla Road": 20 },
    "active_tickets_count": 0,
    "sent_alerts_count": 0,
    "open_resources": [],
    "avg_congestion": 57.875
  },
  "state_after": {
    "traffic_routes": { "G-10 to Blue Area": 85, "Shahrah-e-Faisal": 15, "Margalla Road": 20 },
    "active_tickets_count": 1,
    "sent_alerts_count": 1,
    "open_resources": ["Cooling Centre G-10"],
    "avg_congestion": 48.5
  },
  "estimated_congestion_reduction": 16.2,
  "estimated_response_time_minutes": 8
}
```

| Field | Notes |
|---|---|
| `state_before` / `state_after` | Snapshot of the in-memory `MockSystemState` taken before and after action execution. |
| `estimated_congestion_reduction` | Percentage drop in average congestion across all routes. |
| `estimated_response_time_minutes` | Minimum ETA across all dispatched tickets. |

`recipients_count` is a random integer in `[500, 5000]` per alert.

## Trace

After `/pipeline/run` returns, the full reasoning chain is available at:

```
GET /trace/latest    → full trace with 5 steps + input/output for each
GET /trace/history   → last 10 run summaries
```

Each step in `trace.steps[]`:

```json
{
  "agent": "signal-ingestion-agent",
  "step": "normalise_signal",
  "input": { ... },
  "output": { ... },
  "duration_ms": 3,
  "timestamp": "2026-05-16T13:57:33.629000Z"
}
```

Agent names emitted in order:

1. `signal-ingestion-agent` · step `normalise_signal`
2. `event-detection-agent` · step `detect_crisis`
3. `reasoning-analysis-agent` · step `analyse_crisis`
4. `action-planning-agent` · step `generate_plan`
5. `simulation-agent` · step `execute_actions`

## Error responses

| Status | When |
|---|---|
| `422` | Request body fails Pydantic validation (missing `source` / `text`). |
| `500` | An agent stage raised an unhandled exception. The fallback cache covers the LLM stage, so 500s here usually mean a code-level bug — file an issue. |

The endpoint **never returns partial pipelines**. Either every stage completes (`200`) or the whole call fails (`4xx` / `5xx`).

---

## Stable vs unstable fields

**Stable** — clients may rely on these:
- All field names and enum values listed above
- `run_id` format
- Shape of the 6 top-level keys in `PipelineResult`

**Unstable** — may change without notice:
- Exact wording of `event.explanation`, `analysis.summary`, `alerts_sent[].message`
- Numeric values inside `state_before` / `state_after` (mock data may be retuned)
- The set of fallback cache entries in the Reasoning Agent

## Example clients

### curl
```bash
curl -X POST http://localhost:8000/pipeline/run \
  -H "Content-Type: application/json" \
  -d '{"source":"social","text":"G-10 mein pani bhar gaya hai"}'
```

### Dart / Flutter
```dart
final res = await http.post(
  Uri.parse('$baseUrl/pipeline/run'),
  headers: {'Content-Type': 'application/json'},
  body: jsonEncode({'source': 'social', 'text': userInput}),
);
final result = PipelineResult.fromJson(jsonDecode(res.body));
```

### JavaScript / Web Dashboard
```javascript
const res = await fetch('/pipeline/run', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ source: 'social', text: userInput }),
});
const result = await res.json();
```
